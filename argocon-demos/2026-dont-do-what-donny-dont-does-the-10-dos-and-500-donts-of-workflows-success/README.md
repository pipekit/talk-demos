## The Dos and Don'ts
- Do remember the architecture
- Don't forget that it's all just Kubernetes
- Do observe your cluster
- Do consider log management
- Don't ignore pod resources
- Do set requests and limits
- Don't forget controller and server resources
- Do check defaults in the workflow-controller-configmap
- Do keep things tidy
- Do check your versions
- Don't be insecure
- Do use SSO
- Don't leave Argo open to the internet
- Do get permissions right
- Do set your cluster up for success
- Do prepare for massive scale
- Do set only one controller replica
- Do set a sensible default requeue time
- Don't feel forced to use YAML
- Don't ignore this advice
